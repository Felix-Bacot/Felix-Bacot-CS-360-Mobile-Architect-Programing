# CS-360 Mobile Architecture and Programming

## Inventory Tracker Mobile Application

### Project Overview

This repository contains my completed Project Three submission for CS-360 Mobile Architecture and Programming at Southern New Hampshire University. The project demonstrates the design and development of an Android Inventory Tracker application using Java, SQLite, and Android Studio. The application allows users to create an account, securely log in, manage inventory items, and receive low inventory notifications.

---

# Portfolio Reflection

## Briefly summarize the requirements and goals of the app you developed. What user needs was this app designed to address?

The goal of this project was to design and develop a functional Android mobile application that demonstrates both user-centered design and mobile programming principles. I developed an Inventory Tracker application that allows users to create an account, log in securely, manage inventory items, and monitor inventory quantities. The application was designed to address the need for a simple and efficient way to organize inventory information. By providing an intuitive interface and reliable database functionality, the app enables users to easily track and update their inventory while improving organization and reducing the likelihood of errors.

## What screens and features were necessary to support user needs and produce a user-centered UI for the app? How did your UI designs keep users in mind? Why were your designs successful?

The application included several essential screens, including a login screen, account creation functionality, an inventory management screen, and support for SMS permissions to notify users when inventory levels become low. These screens were designed with simplicity and usability in mind by using clear labels, organized layouts, and logical navigation between tasks. Buttons and input fields were positioned consistently to make the application easy to learn and use. My UI designs were successful because they minimized unnecessary complexity while allowing users to complete their tasks quickly and efficiently.

## How did you approach the process of coding your app? What techniques or strategies did you use? How could those techniques or strategies be applied in the future?

I approached the development process by breaking the project into smaller, manageable components. I first implemented the database functionality using SQLite before developing the login system, inventory management features, and user interface. Each component was tested individually before integrating it into the complete application. This modular development strategy made it easier to identify and resolve issues while maintaining organized and readable code. In future software development projects, I will continue using incremental development, modular programming, and frequent testing because these strategies improve code quality, simplify debugging, and make applications easier to maintain and expand.

## How did you test to ensure your code was functional? Why is this process important, and what did it reveal?

I tested the application throughout development using Android Studio and the Android Emulator. After implementing each feature, I verified that the application compiled successfully and that functions such as user login, account creation, inventory management, database operations, and button interactions worked as expected. Testing was essential because it helped identify issues related to XML layouts, Java code references, database connections, and application logic before the project was completed. This process demonstrated the importance of continuous testing and showed that even small coding errors can significantly affect application performance if they are not identified early.

## Consider the full app design and development process from initial planning to finalization. Where did you have to innovate to overcome a challenge?

One of the most significant challenges during development was integrating multiple application components, including the user interface, SQLite database, Java classes, and Android activity lifecycle. Successfully connecting these components required careful debugging, reviewing error messages, and verifying that XML layouts matched the corresponding Java code. I also adapted my design when resolving issues related to user interface components and database interactions. These challenges strengthened my problem-solving skills and reinforced the importance of persistence, careful planning, and systematic troubleshooting throughout the software development process.

## In what specific component of your mobile app were you particularly successful in demonstrating your knowledge, skills, and experience?

The component that best demonstrates my knowledge and technical skills is the integration of the SQLite database with the inventory management system. Successfully implementing user authentication, inventory storage, and database retrieval required applying object-oriented programming principles, database management concepts, and Android development best practices. I was also successful in creating a clean, user-friendly interface that supports efficient navigation and interaction. This project demonstrates my ability to combine software design, programming, database management, and user-centered design principles into a complete, functional mobile application that satisfies client requirements.
